# e-gov Intake Dashboard

士業の初回相談を、**論点整理・関連法令の洗い出し・確認事項の列挙・初回返信案の下書き** まで一気にまとめるためのローカルアプリだぜ。

相談内容をそのまま貼り付けると、画面上で次のような形に整理されるんな。

- **論点の候補**
- **関連しそうな法令**
- **顧客に追加確認したい事項**
- **慎重なトーンの初回返信案**

法律・許認可・手続の実務に強い人が、**「毎日はコードを書かないけど、大学で少しプログラミングは触ったことがある」** くらいの前提で扱えるように作ってある。

## このプロジェクトが向いている人

- **士業の実務担当者**
  相談メモを整理して、初動を早くしたいやつ向けだぜ。

- **事務所内で試作を触る人**
  エンジニアじゃなくても、ローカルで立ち上げて挙動確認しやすい構成にしてあるんな。

- **生成AIを業務補助に使いたい人**
  ただし、最終判断を AI に丸投げしない前提だぞ。

## できること

このアプリでは、相談文を入力すると主に次の情報を表示するぜ。

1. **Identified Issues**
   相談内容から読み取れる主要な論点を並べる。

2. **Relevant Laws**
   関連候補となる法令や出典を表示する。

3. **Information Checklist**
   顧客や依頼者に追加で確認したい事項を整理する。

4. **Draft Initial Reply**
   すぐ送信するためじゃなく、**人がレビューする前提の下書き** を作る。

5. **追加入力で再整理**
   顧客から追加回答が来たら、その内容を足してもう一度整理できる。

## まず知っておいてほしいこと

このアプリは、**法的結論を断定するためのものじゃない**。

README より先にここを読んどけって話なんだが、設計上も次の考え方を強く持ってるぜ。

- **人間レビュー前提**
  出力はそのまま顧客送付せず、必ず人が確認する。

- **自治体運用や所轄判断を断定しない**
  条文だけで決め打ちせず、実運用の確認を残す。

- **追加ヒアリングを重視する**
  情報不足のまま結論に飛びつかない。

## 画面の流れ

使い方はかなり単純だぜ。

1. 顧客名を入れる
2. 相談内容を貼り付ける
3. `Start Analysis` を押す
4. 整理された結果を読む
5. 必要なら追加情報を入れて再分析する

初回デモ用に、飲食店営業・建設業許可・在留資格みたいな**プリセット例**も入ってるんな。

## 技術的には何をしているのか

コードに慣れてない人向けに、難しい言い方を避けてざっくり説明するとこうだぜ。

- **画面側**
  `React` で入力画面とダッシュボード画面を作ってる。

- **裏側の処理**
  `Express` ベースのサーバーが、相談文を受け取って整理処理を実行する。

- **AI 連携**
  `OpenRouter` 経由でモデルを呼び出して、下書き生成や構造化を補助する。

- **法令・業務向けの整理ロジック**
  MCP 形式のツール群として、論点抽出や返信文下書きの流れをまとめてる。

## 動かすために必要なもの

最低限これだけあればいい。

- **Node.js 22系**
- **npm**
- **OpenRouter の API キー**

## セットアップ手順

### 1. パッケージを入れる

プロジェクトのルートで次を実行する。

```bash
npm install
```

### 2. 環境変数ファイルを作る

見本ファイル `.env.example` をコピーして、実際に使う `.env.local` を作る。

```bash
cp .env.example .env.local
```

### 3. API キーを設定する

`.env.local` を開いて、少なくとも次を設定するんだ。

```env
OPENROUTER_API_KEY="あなたのAPIキー"
```

必要に応じて、モデル名や API の URL も `.env.local` で調整できるぜ。

### 4. 開発サーバーを起動する

```bash
npm run dev
```

起動したら、ブラウザで次を開けば使える。

```text
http://localhost:3000
```

## よくあるつまずきどころ

- **API キー未設定**
  AI 呼び出し部分でエラーになる。まず `.env.local` を見直せ。

- **`npm install` をしていない**
  依存パッケージが足りず、起動時に落ちる。

- **出力をそのまま正解だと思い込む**
  このアプリは整理補助だ。最終判断は必ず実務側で確認しろよな。

## この README を読むだけで分からなくても大丈夫な用語

毎日コードを書かない人向けに、最低限だけ置いとくぞ。

- **フロントエンド**
  画面に見えている部分のことだぜ。

- **バックエンド / サーバー**
  画面の裏で処理している部分だ。

- **環境変数**
  API キーみたいな、コードに直接ベタ書きしたくない設定値のことだな。

- **ローカルで動かす**
  インターネット上に公開せず、自分のパソコン上で試すことだ。

## 注意事項

- **法的助言の最終版として使わないこと**
- **顧客送付前に必ずレビューすること**
- **自治体・所轄・最新法令の確認を省略しないこと**
